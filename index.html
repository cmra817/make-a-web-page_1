Lesson 1: Basic Concepts

Web Browsers, Internet, and Servers

Web browsers are what the end users see. The Internet is a compilation of HTML documents that communicate with each other. Servers are where the database are stored.

HTML

HTML stands for Hypertext Markup Language. It is the language that runs computers. It tells what the computer should do, say and look like. To have this much control on the computer is a really powerful knowledge!

Inline vs. Block

Inline are tags that are displayed without line breaks. Block tags have an invisible box around them, and normally start and end with a new line.

The following are inline elements: a, br, em, b, img, span
The following are block elements: p, div, and whaterver form is. (We haven't covered that yet.)

Lesson 2: Creating a Structured Document

Document Object Model (DOM)

DOM is a tree-structure or a diagram that represents the hierarchical nature of a structure in a graphical form. Once you know how a DOM works or how it looks like, you can pretty much deconstruct a website just by looking at it and identify the different elements within a block of elements

HTML is the language of the Web. We use tags to define HTML because the browser turns them into elements in a tree for DOM.

CSS

CSS is the styling sheet that changes how elements are displayed on the page.

Boxify Design

Developed pages consist of a whole bunch of boxes. Every single element in a Web page is boxed in.
You define boxes with div tags and you use class tags to label what the boxes are. I have a feeling this is so that it's easier when you apply the style in CSS.

Lesson 3: Adding CSS Style and HTML Structure

CSS

CSS stands for Cascading Style Sheets. It avoids repetition. CSS provides efficiency in design and updates. With CSS, we are able to create rules, and apply those rules to many elements within the website. This avoid repetition of rules and increases efficiency.
Another way to avoid repetition is by using the inherit value. Every now and then there will be times when you have to repeat the same value for a particular property in several CSS rule. Sometimes doing so is necessary, but there are some situations when you use the inherit value to avoid repeating yourself.

Inheritance

Inheritance is a key feature in CSS, it relies on the ancestor-descendent relationship. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to descendents.

Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing us to write less CSS.
Box related elements can be inherited, but not box-related elements.

Div vs. Span

Div are block elements, whereas Spans are in-line elements. When an element is within a div, it is enclosed inside a box. However, when an element is within a span, it is not necessarily inside a box. Instead, it separates it from the rest of the content without grouping it within its own individual box. So any element you want to change without changing the layout should use a span.

Class vs. Id

ID is used to uniquely identify one single element in your entire website. There should only be one type of ID in the website. You use the rule #id in CSS.
Classes are used to style multiple elements that have similar styling properties. You use the rule .class in CSS.

Flex Box Layout

Flex box layout lex is a shorthand property specifying the ability of a flex item to alter its dimensions to fill available space. Flex items can be stretched to use available space proportional to their flex grow factor or their flex shrink factor to prevent overflow.
If you want elements to be next to each other, use flex. Divs are block elements, so by default they take up the entire width of a page. Adding the rule display: flex to CSS will override this behavior and let divs appear next to each other.

Percentage vs. Pixels in Sizing Layout

You can either use % or px to define the size of your divs. The difference between the two is that when you use %, the size of your divs will always be relative to the size of your window. For example, if you were you shrink or expand your window, your divs will be responsive and will change its size to meet the percentage size relative to the window.
Pixels, on the other hand, are absolute values that won't change its size relative to the size of the window.
